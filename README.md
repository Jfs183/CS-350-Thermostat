CS-350 Module Eight Journal – Portfolio Submission

Project Summary

For this project, I built a thermostat system using a Raspberry Pi, integrating hardware and software. I used buttons to switch between OFF, HEAT, and COOL modes, LEDs to indicate the system's status, and an LCD screen to display the temperature and set point. My main goal was to design and build an embedded system that showed how software and hardware can work together.

What I Did Well

I think I did a good job troubleshooting both hardware and software problems. For example, I identified and fixed issues with the LED wiring by understanding how the circuits worked. I also set up a state machine to control the system and ensured the LEDs and LCD displayed the correct information for each mode.

Where I Could Improve

I could improve my time management and planning. I ran into some problems that made me go back and fix earlier mistakes, like indentation errors and wiring issues. Also, since I didn’t have a third button, testing all the system states was harder, and I had to change how I tested.

Tools and Resources Added to My Support Network

During this project, I used Python documentation, Raspberry Pi guides, and step-by-step troubleshooting to solve problems. I also looked up solutions online and used various debugging methods to better understand both the code and the hardware. These resources will help me in future projects too.

Transferable Skills

This project helped me develop several transferable skills, including:

* Debugging both hardware and software issues
* Understanding and implementing state machines
* Writing clean and structured Python code
* Integrating hardware components with software systems
* Problem-solving using a systematic approach

Maintainability, Readability, and Adaptability

I kept the project maintainable and easy to read by organizing my code into functions and using clear names. The state machine structure makes it simple to understand and change the system. By keeping the hardware logic separate from the display logic, it’s also easier to update or improve things later. These choices make the system flexible for future upgrades or new hardware.
